# 功能清单与产品定位

## 一、产品定位

### 核心主张

> **SSH-native · Zero-agent · Desktop-first**
>
> 面向技术型个人开发者的 All-in-One 服务器管理桌面应用。
> 下载即用，无需公网服务器。在 Mac / Linux 本机运行，管理所有远程服务器，系统托盘常驻，告警直达原生通知。
> 媲美 Coolify 的部署便利，超越 1Panel 的运维能力，全部通过 SSH 实现，不在被管服务器安装任何 Agent。

### 两种运行模式

| 模式 | 适合场景 | 说明 |
|------|---------|------|
| **桌面模式**（推荐） | 个人开发者 | 在 Mac/Linux 本机运行，系统托盘常驻，原生通知告警，浏览器访问 localhost |
| **服务器模式** | 团队共用 / 需要 Webhook | 部署在公网服务器，支持 Push-to-Deploy Webhook，24/7 持续监控 |

### 与主流竞品的差异

| 维度 | 宝塔 | 1Panel | Coolify | **ServerHub** |
|------|------|--------|---------|--------------|
| 被管服务器要求 | 安装 Agent | 安装 Agent | 安装 Agent | **只需 SSH 访问** |
| 面板运行位置 | 需公网服务器 | 需公网服务器 | 需公网服务器 | **本机 Mac/Linux 即可** |
| 部署方式 | Python + DB | Docker | Docker | **单二进制** |
| 面板内存占用 | 200MB+ | 500MB+ | 2GB+ | **< 100MB** |
| WireGuard 内网服务器 | 困难 | 困难 | 不支持 | **原生支持** |
| 强制 Docker | 否 | 是 | 是 | **否** |
| 裸金属运维能力 | 完整 | 弱 | 无 | **完整** |
| Git 部署 | 否 | 弱 | 完整 | **完整** |
| 告警通知 | 站内 | 站内 | 站内 | **原生系统通知（桌面模式）** |
| 数据隐私 | 有争议 | 开源 | 开源 | **完全自主** |

### 目标用户

- 有 3-5 台 Linux 服务器（含内网机器）的个人开发者或小团队
- 不想在每台服务器上安装额外软件，也不想为"面板"再维护一台公网服务器
- 需要同时具备部署便利性（类 Coolify）和服务器运维能力（类 1Panel）
- 对数据安全有顾虑，不信任第三方面板的数据采集行为

## 三、不在 MVP 范围内（明确不做）

| 功能 | 原因 |
|------|------|
| 多用户管理界面 | 架构已预留（users 表），UI 暂不开放，`allow_register: false` |
| 插件 / 扩展系统 | 增加架构复杂度，超出个人工具定位 |
| 软件商店（宝塔式） | 需要 Agent，违背 SSH-native 定位 |
| DNS 管理 | 超出服务器运维范畴 |
| 账单 / 计费系统 | 个人工具不需要 |
| 手机原生 App | Web 响应式布局已足够移动端访问 |
| 多租户 SaaS | 超出定位 |
| Push-to-Deploy Webhook（桌面模式） | 无公网端点；可手动触发部署或使用服务器模式 |

## 四、多用户升级路径

当前单用户设计与多用户的唯一差异在于：

```yaml
# config.yaml
security:
  allow_register: false   # 改为 true 即开放注册
```

数据库层已完整支持多用户（users 表、JWT 携带 user_id、资源关联 created_by），升级只需：
1. 将 `allow_register` 改为 `true`
2. 开放前端用户管理页面入口
3. 按需添加 RBAC 权限检查中间件

无需重构任何业务逻辑。

## 五、v2.0 交互架构演进

> 详细设计文档: [docs/interaction-redesign.md](./interaction-redesign.md)
> 实施计划: `.zcf/plan/current/双视角交互架构重构.md`

### 核心变更

从 v1.0 的"工具优先"扁平导航演进为"双视角统一"架构：

| 维度 | v1.0 | v2.0 |
|------|------|------|
| 导航模型 | 12 个平级工具页 | 应用视角 + 服务器视角（树形导航） |
| 服务器上下文 | 每页独立选择 | 从路由参数自动获取 |
| 应用聚合 | 无 | Application 模型关联所有资源 |
| Dashboard | 服务器指标墙 | 运维工作台（待办 + 活动流） |
| Nginx 概念 | 扁平"网站"页 | 服务器级"网关"+ 应用级"域名" |

### 新增模块

- **应用管理**: Application 模型 + 聚合视图（概览/域名/服务/部署/日志/DB/环境变量）
- **Nginx 网关**: 服务器级别的路由表总览
- **运维工作台**: 待处理事项 + 活动流
- **扫描导入**: 自动读取 Nginx 配置 → 批量创建应用
